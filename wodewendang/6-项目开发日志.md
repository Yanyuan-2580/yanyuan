## v1.0 — 项目初始化

**时间**: 2026年4月
**标签**: `v1.0`

### 完成内容

#### 前端 (client/)
- 使用 `npm create vite@latest` 创建 Vue 3 + TypeScript 项目脚手架。
- 集成 Tailwind CSS、Vue Router、Pinia 状态管理。
- 创建基础页面结构：首页 Home、登录 Login、注册 Register、AI对话 Chat。
- 搭建路由框架，配置路由守卫（根据 localStorage accessToken 判断登录状态）。
- 创建 `api/modules/` 目录结构，封装 axios 请求实例（baseURL + 拦截器）。
- 引入 Lucide Icons (`lucide-vue-next`) 作为统一图标库。

#### 后端 (server/)
- 使用 NestJS CLI 创建项目脚手架。
- 配置 TypeORM 连接 MySQL 数据库，定义 `database.config.ts`。
- 配置 Mongoose 连接 MongoDB 数据库。
- 创建 User 实体（TypeORM）：id、username、password(加密)、email、phone、avatar、status、createdAt、updatedAt。
- 创建 Admin 实体（TypeORM）。
- 实现 JWT 认证：accessToken（2h）+ refreshToken（7d）。
- 实现用户模块 API：`POST /api/v1/user/register`、`POST /api/v1/user/login`、`GET /api/v1/user/profile`。
- 引入 DeepSeek API，实现基础 AI 对话（非流式）：`POST /api/v1/chat/send`。
- 创建 AiSession 实体（TypeORM）和 ChatMessage schema（MongoDB）。

#### 基础设施
- `.env` 环境变量配置：数据库连接、JWT 密钥、DeepSeek API Key。
- 统一 API 响应格式：`{ code, message, data, timestamp }`。
- 统一分页格式：`{ list, total, page, pageSize, totalPages }`。

### 技术决策
- 用户关系数据存 MySQL（TypeORM），聊天消息存 MongoDB（Mongoose）—— 消息数据量大、结构半结构化。
- JWT 双 Token 机制：accessToken 短期有效减少泄露风险，refreshToken 实现无感刷新。
- 前端不引入 UI 库，纯 Tailwind CSS 手写样式 —— 为了完全掌控视觉风格。

### 文件清单（关键文件）
- `client/src/main.ts`、`client/src/App.vue`、`client/src/router/index.ts`、`client/src/api/request.ts`
- `client/src/pages/home/index.vue`、`client/src/pages/auth/login.vue`、`client/src/pages/auth/register.vue`、`client/src/pages/chat/index.vue`
- `server/src/main.ts`、`server/src/app.module.ts`、`server/src/config/database.config.ts`
- `server/src/database/entities/user.entity.ts`、`server/src/database/entities/admin.entity.ts`、`server/src/database/entities/ai-session.entity.ts`
- `server/src/database/schemas/chat-message.schema.ts`
- `server/src/modules/user/user.module.ts`、`server/src/modules/user/user.controller.ts`、`server/src/modules/user/user.service.ts`
- `server/src/modules/chat/chat.module.ts`、`server/src/modules/chat/chat.controller.ts`、`server/src/modules/chat/chat.service.ts`
- `server/src/common/guards/jwt-auth.guard.ts`

### 问题记录
1. **跨域问题**：前端 Vite 开发服务器 (5173) 请求后端 (3000) 时 CORS 报错。解决方案：在 NestJS main.ts 中启用 `app.enableCors()`。
2. **MongoDB 连接超时**：本地 MongoDB 服务未启动。解决方案：添加连接状态检测和重试机制。

## v1.1 — SSE流式对话 + 多会话管理

**时间**: 2026年4月
**标签**: `v1.1`

### 完成内容

#### 前端 (client/)
- 聊天页面改造：支持 SSE 流式接收 AI 回复，打字机效果逐字显示。
- 实现 SSE 断线重连机制（指数退避，最大重试 5 次）。
- 增加非流式回退：SSE 连接失败时自动降级为普通 HTTP 请求。
- 会话列表页面：显示历史会话、新建会话、删除会话、重命名会话。
- 聊天页面顶部显示当前会话标题，支持切换会话。
- 空状态处理：显示欢迎引导文案和快捷话题入口。

#### 后端 (server/)
- 实现 SSE 端点：`GET /api/v1/chat/messages/stream/:sessionId`。
- SSE 流式转发 DeepSeek API 的流式响应。
- 会话 CRUD API：`GET /api/v1/chat/sessions`、`POST /api/v1/chat/sessions`、`PUT /api/v1/chat/sessions/:id`、`DELETE /api/v1/chat/sessions/:id`。
- 消息分页查询：`GET /api/v1/chat/messages?sessionId=xxx&page=1&pageSize=20`。
- 会话级别 Token 计数和限制。

### 技术细节
- SSE 实现使用 NestJS `@Sse()` 装饰器 + RxJS Observable。
- 前端使用 `EventSource` API 接收 SSE，Vue 3 `ref` 驱动逐字更新。
- 会话管理：AiSession 表（MySQL）存会话元数据，ChatMessage 集合（MongoDB）存消息体。

### 问题记录
1. **SSE 流中文乱码**：DeepSeek 返回的流中中文字符被截断。解决方案：前端 Buffer 合并后再解码。
2. **会话切换时消息残留**：切换 sessionId 时未清空旧消息。解决方案：watch sessionId 变化时重置 messages 响应式变量。

## v2.2 — 风控系统 + 通知 + 评论

**时间**: 2026年5月
**标签**: `v2.2`

### 完成内容

#### 风控系统
- **高危词库**：35+ 高危关键词（自伤/自杀/暴力/违法相关），使用正则匹配检测。
- **中危词库**：15+ 中危关键词（焦虑/抑郁/失眠严重程度相关），用于风险评估。
- **风险升级追踪**：同一用户短时间内多次触发中危词，自动升级为高危风险记录。
- **检测覆盖点**：AI 聊天消息内容、日记内容、评论内容。
- **风控服务**：`server/src/shared/risk-control/risk-control.service.ts`。

#### 通知系统
- Notification 实体（TypeORM）：id、userId、type、title、content、isRead、relatedId、createdAt。
- 通知类型：系统通知、评论回复、风险提醒、测评结果。
- 通知 API：`GET /api/v1/notification/list`、`GET /api/v1/notification/unread-count`、`PUT /api/v1/notification/read/:id`、`PUT /api/v1/notification/read-all`。
- 前端未读计数轮询：每 30 秒查询一次未读通知数量，在底部导航栏显示红点。
- SMS/Email 发送接口（预留，当前仅 DB 通知）。

#### 评论系统
- Comment 实体（TypeORM）：id、userId、articleId、parentId（嵌套回复）、content、createdAt。
- 评论嵌套：支持一级评论 + 二级回复，前端递归渲染。
- 评论风控检查：提交评论时自动过风控检测。
- 评论 API：`GET /api/v1/comment/list?articleId=xxx`、`POST /api/v1/comment`、`DELETE /api/v1/comment/:id`。

### 技术细节
- 风控检测使用责任链模式：消息 → 高危词检测 → 中危词检测 → 上下文分析 → 风险等级判断。
- 评论嵌套在前端使用 Vue 递归组件渲染。

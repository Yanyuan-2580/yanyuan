## 一、引言

### 1.1 编写目的

本需求规格说明书（Software Requirements Specification, SRS）旨在完整、清晰、无歧义地定义"心理健康AI助手"项目的全部功能需求和非功能需求。本文档面向以下读者群体：

- **开发团队**：作为系统设计、编码实现和单元测试的直接依据
- **测试团队**：作为编写测试用例和执行验收测试的标准参照
- **UI/UX设计师**：理解功能全貌，确保设计方案覆盖所有用户场景
- **内容顾问**：了解系统能力边界，指导心理健康内容的策划和审核
- **项目经理**：管理需求变更、把控项目范围和评估进度影响
- **投资方和决策者**：了解产品的完整功能蓝图和交付范围

本文档采用"用户故事 + 功能说明 + 输入输出描述"的方式组织，确保每个需求条目具有可验证性和可追踪性。

### 1.2 产品概述

"心理健康AI助手"是一个基于人工智能技术的综合性数字心理健康服务平台。产品以"AI对话陪伴"为核心引擎，整合情绪日记、心情追踪、冥想放松、心理知识库、专业测评、视频咨询和危机热线等功能模块，为用户提供从日常情绪管理到专业心理成长的全旅程支持。

产品采用极简治愈轻奢风的设计语言（雾松青 #577f63 / 柔烟蓝 #8b95be / 奶杏白 #faf8f5），通过温暖、宁静、包容的视觉体验，降低用户使用心理服务时的心理防御，营造安全、信任的疗愈空间。

当前产品形态为响应式Web应用（用户端 + 管理后台），计划在后续版本中以微信小程序形态覆盖移动端场景。

## 二、功能需求

### 2.1 用户认证与账户管理模块

**用途说明**：提供用户注册、登录、密码管理和账户基础信息维护功能，是整个系统的入口和用户身份管理中枢。采用JWT双Token机制确保安全性和用户体验的平衡。

**用户输入**：
1. 注册：手机号/邮箱 + 密码（8位以上，含大小写字母和数字）+ 验证码（短信/邮箱）+ 同意用户协议和隐私政策
2. 密码登录：手机号/邮箱 + 密码
3. 验证码登录：手机号/邮箱 + 验证码
4. 忘记密码：手机号/邮箱 + 验证码 + 新密码 + 确认新密码
5. 个人资料编辑：昵称（1—20字符）、头像（上传图片，限制2MB）、个性签名（可选，最多200字符）、性别（可选）、出生日期（可选）
6. 密码修改：旧密码 + 新密码 + 确认新密码
7. 账户注销：确认注销 + 密码验证 + 阅读注销后果说明

**预期输出**：
1. 注册成功：自动登录 + 返回用户基本信息 + accessToken + refreshToken，跳转至首页
2. 登录成功：返回用户基本信息 + accessToken（2小时有效）+ refreshToken（7天有效），跳转至首页或来源页面
3. Token刷新：accessToken过期时自动使用refreshToken刷新，对用户透明
4. 验证码发送：60秒发送冷却倒计时，防止频繁请求
5. 密码修改成功：退出所有设备登录（或仅当前设备修改），强制重新登录
6. 账户注销：30天冷静期后彻底删除账户数据，冷静期内可撤回注销申请
7. 错误处理：密码错误（返回剩余尝试次数）、验证码错误（返回重新获取提示）、账户被冻结（返回联系客服指引）

### 2.2 AI对话陪伴模块

**用途说明**：核心模块，利用大语言模型（DeepSeek API）为用户提供7x24小时的AI心理陪伴对话服务。AI以共情、非评判、积极引导的方式进行对话，帮助用户表达情绪、梳理思路、获得情感支持。同时内建风险识别机制，在高危情境下自动触发保护性干预。

**用户输入**：
1. 文本消息（每条最长2000字符）
2. 会话管理操作：创建新会话、切换会话、重命名会话、删除会话
3. 会话列表浏览：查看所有历史会话及最后一条消息预览

**预期输出**：
1. AI回复以流式（SSE）方式逐字呈现，首字响应时间 < 3秒
2. 每条AI回复附带免责声明："本内容由AI生成，仅供参考，不能替代专业心理诊断和治疗"
3. 对话历史持久化保存，用户可随时回溯历史对话
4. 当系统检测到用户输入包含高危关键词（35+高危词库）时：
   - AI回复转为预设安全引导内容
   - 自动推送危机干预热线信息（如希望24热线：400-161-9995）
   - 在用户端以关怀提示语展示，不制造恐慌
   - 后台标记为高风险对话，供管理员复核
5. 当系统检测到中危关键词（15+中危词库）时：
   - AI回复加强积极引导和心理教育元素
   - 记录风险标记，如连续命中则升级为高危处理
6. 会话管理操作的即时反馈（创建/删除/切换均 < 500ms）
7. 当AI API服务不可用时自动降级：显示预设引导内容 + 推荐自助工具（冥想/日记/知识库）

### 2.3 情绪日记模块

**用途说明**：提供结构化的日记写作工具，结合AI自动情绪分析能力，帮助用户通过书写进行情绪表达和自我反思。日记支持私密和公开两种模式，满足不同用户的分享需求。

**用户输入**：
1. 创建日记：标题（必填，最多100字符）+ 正文内容（必填，支持换行和段落，最长10000字符）+ 心情标签（可选，从预设列表中选择）+ 日期（默认当天，可回溯选择）+ 公开/私密切换
2. 编辑日记：修改标题、正文、心情标签、日期、公开/私密状态
3. 日记列表浏览：按时间倒序排列、按心情标签筛选、按关键词搜索
4. 日记详情查看：查看单篇日记完整内容和AI分析结果
5. 删除日记：确认删除操作

**预期输出**：
1. 日记保存后，系统自动调用AI分析日记内容的情绪倾向，返回：
   - 主导情绪标签（如：焦虑、平静、悲伤、愤怒、喜悦、困惑等）
   - 情绪强度评分（1—10）
   - 一段2—5句话的AI共情回应和积极引导建议
   - 分析耗时展示（通常 < 5秒）
2. 日记列表按时间分组展示（今天/昨天/本周/更早），每条显示标题摘要、日期、心情标签、前50字预览
3. 公开日记在"社区"板块可见，其他用户可浏览和互动
4. 日记统计面板：本周写作天数、总日记篇数、情绪标签分布饼图
5. 搜索功能支持标题和正文关键词检索
6. 删除操作需二次确认，删除后不可恢复

### 2.4 心情追踪模块

**用途说明**：提供轻量、快速的心情记录工具，用户可在10秒内完成一次心情打卡。系统通过持续的心情数据积累，生成可视化的心情变化趋势，帮助用户觉察自身情绪波动规律。

**用户输入**：
1. 快速记录：心情评分（滑动条1—10分，1代表"非常糟糕"，10代表"非常好"）+ 情绪标签（多选，如开心/焦虑/平静/疲惫/压力/期待/孤独/感恩等）+ 心情笔记（可选，最多500字符）
2. 查看历史：按日/周/月切换心情记录列表和统计图表
3. 提醒设置：每日心情记录提醒开关 + 提醒时间选择

**预期输出**：
1. 记录保存后即时反馈：心情评分动画展示 + 情绪标签可视化 + 简短鼓励语
2. 日视图：当天每次心情记录的时间线和评分变化
3. 周视图：折线图展示7天心情平均分变化趋势 + 本周情绪标签分布雷达图
4. 月视图：30天心情热力图 + 月度平均分 + 最高/最低日标注 + 月度情绪词云
5. 统计数据：周期平均心情分、标准差（情绪稳定性指标）、记录总天数、连续记录天数
6. 提醒功能在设定时间推送浏览器通知（需用户授权）

### 2.5 冥想放松模块

**用途说明**：提供多种类别的冥想引导内容和计时工具，帮助用户通过正念练习缓解压力、改善睡眠、提升专注力。通过打卡记录和连续天数统计激励用户养成冥想习惯。

**用户输入**：
1. 浏览冥想列表：按分类筛选（减压/睡眠/专注/正念/身体扫描）
2. 选择冥想课程：查看详情（标题、描述、时长、难度级别、背景音频类型）
3. 开始冥想：选择自定义时长（默认5/10/15/20/30分钟可选）或使用预设时长
4. 冥想过程控制：暂停/继续/结束/跳过引导
5. 记录冥想：冥想结束后确认记录时长
6. 查看冥想历史和个人统计

**预期输出**：
1. 冥想列表：卡片式展示，每张卡片包含封面图、标题、时长标签、难度标签
2. 冥想详情页：完整描述、引导方式说明、适合场景、用户评价（预留）
3. 冥想进行中界面：计时器（倒计时/正计时切换）+ 呼吸动效引导 + 背景音播放 + 进度条
4. 冥想结束界面：本次冥想时长 + 鼓励成就语 + 分享按钮（可选）+ 连续冥想天数展示
5. 历史记录列表：按日期分组，显示每次冥想的时间、时长、类型
6. 统计面板：累计冥想时长、总次数、连续天数、月度统计柱状图、类型分布饼图
7. 成就徽章：首次冥想、连续7天、累计100分钟、累计30次等里程碑成就（Phase 6）

### 2.6 心理知识库模块

**用途说明**：提供系统化、科学化的心理健康知识文章库。所有文章按分类组织，经过心理专业人员审核把关，确保内容的科学性。支持用户搜索、浏览、点赞、收藏和评论互动。

**用户输入**：
1. 文章浏览：按分类筛选（焦虑管理/抑郁科普/人际关系/亲密关系/自我成长/职场心理/青少年心理/睡眠健康等）+ 按推荐/最新/最热排序
2. 文章搜索：输入关键词进行标题和正文搜索
3. 查看文章详情：完整文章内容 + 相关文章推荐
4. 文章互动：点赞（可取消）、收藏（可取消）、发表评论（纯文本，最长1000字符）、回复他人评论、点赞评论
5. 个人收藏夹：查看已收藏的文章列表

**预期输出**：
1. 文章列表：卡片或列表形式，展示封面图、标题、摘要（前150字）、作者、发布时间、阅读量、点赞数、分类标签
2. 文章详情页：Markdown渲染的完整正文 + 文章底部的点赞/收藏/分享按钮 + 评论区
3. 搜索功能：搜索结果按相关度排序，高亮显示匹配关键词，支持搜索结果的分页加载
4. 评论系统：支持一级评论和二级回复（嵌套一层），每条评论经风控扫描后发布，包含评论者昵称、头像、时间、点赞数
5. 点赞/收藏操作即时反馈（按钮状态切换动画，< 300ms），数量实时更新
6. 相关文章推荐：基于当前文章分类和标签，推荐3—5篇相关文章
7. 收藏夹支持分页浏览和取消收藏操作

### 2.7 心理测评模块

**用途说明**：提供标准化的心理量表测评工具，帮助用户客观了解自己的心理状态。基于JSON题库架构，支持灵活扩展多种量表。系统自动计分、分级，并由AI生成个性化的结果解读和建议。

**用户输入**：
1. 浏览测评列表：查看可用的心理测评量表（列表展示名称、简介、题目数、预计耗时）
2. 开始测评：逐题作答，每道题选择对应选项（单选题，支持前进/后退修改）
3. 提交测评：完成所有题目后确认提交
4. 查看测评结果：包含分数、等级、详细解读和建议
5. 查看历史结果：可对比不同时间的测评结果变化

**预期输出**：
1. 测评列表页：每种量表以卡片展示，包含量表名称、简短说明、题目数量、预计时长（如"共9题，约3分钟"）
2. 测评答题界面：进度条 + 题目编号 + 题目文本 + 选项列表（点击选择）+ 上一题/下一题按钮 + 答题完成确认提示
3. 测评结果页（自动评分后展示）：
   - 维度分数（如PHQ-9的总分0—27分，按5/10/15/20分界为无/轻度/中度/中重度/重度）
   - 结果等级标签（颜色编码：绿/黄/橙/红）
   - AI生成的个性化解读（2—3段，包括：分数含义解释、当前状态的客观描述、实用建议）
   - 免责声明："本测评结果仅供参考，不能作为临床诊断依据。如有困扰，请寻求专业心理帮助"
   - 推荐相关功能：如结果偏焦虑，推荐使用冥想模块；偏抑郁，推荐咨询专业热线
4. 高危结果处理（如PHQ-9 >= 20分或自杀相关题目高危应答）：
   - 结果页额外展示危机干预热线和一键拨打引导
   - 温和建议寻求专业帮助
   - 后台标记为高风险测评记录
5. 历史结果页：按时间倒序列出已完成测评，可点击查看完整结果，支持简单的前后对比
6. 未完成测评保存：用户退出答题时提示是否保存进度，下次进入可继续作答

### 2.8 视频咨询模块

**用途说明**：提供视频会话的房间管理功能，为用户与咨询师（或同伴）的在线视频交流提供基础支持。本模块在Phase 3完成基础框架，为后续对接专业视频咨询师资源做准备。

**用户输入**：
1. 创建视频房间：房间名称 + 房间描述（可选）
2. 加入已有房间：输入或选择房间
3. 房间内操作（基础）：开启/关闭摄像头、开启/关闭麦克风、离开房间
4. 房间列表：查看可用房间

**预期输出**：
1. 房间创建成功：生成唯一房间号（可复制分享）+ 显示房间状态（等待中/进行中/已结束）
2. 加入房间：建立视频连接（基于WebRTC）+ 显示参与者列表
3. 视频通话界面：本端视频窗口 + 远端视频窗口 + 控制栏（摄像头/麦克风/挂断）
4. 房间列表：显示当前活跃的房间信息（名称、人数、创建时间）
5. 房间结束后自动更新状态为"已结束"

### 2.9 危机热线模块（内嵌功能）

**用途说明**：聚合全国权威的心理危机干预热线资源，在用户情绪危机时提供一键求助通道。本模块作为系统级的保护网，并非独立页面，而是渗透在AI对话、测评结果等高风险场景中。

**用户输入**：
1. 查看热线列表：浏览各级各类心理援助热线
2. 一键呼叫：点击热线号码直接拨出（移动端）或复制号码（桌面端）
3. 紧急联系人设置（可选）：预设一个紧急联系人电话

**预期输出**：
1. 热线列表页：全国/省/市级心理援助热线，每条包含机构名称、热线号码、服务时间、服务人群说明
2. 高危场景自动触发：AI对话检测到高危、测评结果高危时，自动在界面展示热线卡片
3. 热线卡片：包含热线号码（大号字体突出显示）、机构名称、"立即呼叫"按钮、温和的关怀文案
4. 紧急联系人功能：设置成功后，高危场景提供"通知紧急联系人"快捷操作

### 2.10 用户中心模块

**用途说明**：汇集用户个人设置、数据管理、通知查看、帮助中心等功能，是用户管理个人账户和应用体验的控制台。

**用户输入**：
1. 个人资料编辑（见2.1模块）
2. 隐私设置：日记默认可见范围、是否允许搜索到我的公开日记、是否接收推送通知
3. 通知管理：按类型筛选通知（系统通知/互动通知/提醒通知）、标记单条已读/全部已读
4. 数据管理：导出个人数据（JSON格式）、删除账户
5. 帮助与反馈：查看常见问题FAQ、联系客服、提交反馈
6. 关于：查看应用版本信息、用户协议、隐私政策

**预期输出**：
1. 个人资料页：头像 + 昵称 + 签名 + 注册天数 + 各模块使用统计概览（日记篇数、冥想时长、心情记录天数、测评次数）
2. 隐私设置：开关状态清晰展示，设置实时保存
3. 通知列表：按时间倒序展示，未读通知高亮显示，显示相对时间（如"3分钟前""昨天"）
4. 未读通知计数：页面标题栏和底部导航栏的角标数字
5. 数据导出：点击导出后生成JSON文件（包含用户基本信息和各模块数据），提供下载链接
6. 帮助页面：分类FAQ + 搜索功能
7. 关于页面：版本号、更新日志、开源许可信息

### 2.11 管理后台模块

**用途说明**：为平台管理员提供全面的数据管理、内容审核和风险监控工具。管理后台独立于用户端，采用Element Plus组件库，以数据仪表盘为首页。

**用户输入**：
1. 管理员登录：管理员账号 + 密码（独立于用户端认证体系）
2. 仪表盘：查看核心数据指标
3. 用户管理：搜索用户、查看用户详情、修改用户状态（正常/禁用）、查看用户各模块数据
4. 文章管理：创建文章（Markdown编辑器）、编辑文章、审核文章（发布/待审核/下架）、分类管理
5. 对话管理：查看用户AI对话记录、标记/取消标记风险对话、按风险等级筛选对话
6. 日记管理：查看用户日记列表、删除不当日记内容
7. 风险记录：查看所有风控命中记录（高危/中危）、按风险等级/时间/用户筛选、标记处理状态（未处理/已处理/无需处理）
8. 数据统计：按时间段查看各模块数据统计（新增用户数、对话总数、日记篇数、测评次数等）
9. 操作日志：查看管理员操作记录（谁在什么时间做了什么操作），用于审计

**预期输出**：
1. 仪表盘首页：数据卡片展示核心指标（总用户数/今日新增/今日对话数/今日风险命中数）+ 趋势图表（本周用户增长曲线/对话量曲线）+ 最近风险记录列表
2. 用户管理表格：分页表格（用户ID/昵称/手机号/注册时间/状态/操作按钮），支持搜索和筛选
3. 用户详情：用户基本信息 + 各模块使用统计 + 最近对话摘要 + 最近日记列表 + 风险记录
4. 文章管理：文章列表表格 + Markdown编辑/预览双栏界面 + 分类树形管理
5. 对话记录：对话列表 + 点击查看完整对话记录 + 风险标记高亮显示（绿色/橙色/红色）+ 管理员注记
6. 风险记录表格：分页表格（风险等级色标/用户/触发内容摘要/模块/时间/处理状态/操作），支持筛选和批量处理
7. 数据统计：时间范围选择器 + 统计图表（用户增长折线图/对话量柱状图/功能使用分布饼图/风险趋势图）+ 数据导出为表格
8. 操作日志表格：分页表格（操作时间/管理员/操作类型/操作对象/详情/IP地址），可按时间范围和管理员筛选

### 3.1 性能需求

|------|--------|---------|
| API响应时间（常规请求） | P95 < 500ms | APM工具监控 |

| 页面首次加载时间 | < 3秒（4G网络下） | Lighthouse Performance评分 |
| 系统并发用户数 | 支持5000人同时在线 | 压力测试 |
| AI对话并发数 | 支持500路并发SSE连接 | 压力测试 |
| 数据库查询时间 | 单次查询 < 100ms（有索引） | 慢查询日志 |
| 缓存命中率 | > 90% | Redis统计 |
| 前端打包体积 | 主包 < 500KB gzipped | Vite build分析 |
| 服务器CPU使用率 | 峰值 < 70% | 服务器监控 |
| 系统可用性（SLA） | 99.5%（月度） | 宕机时间统计 |

### 3.2 兼容性需求

| 平台 | 兼容范围 |
|------|---------|
| 桌面浏览器 | Chrome 90+, Edge 90+, Firefox 88+, Safari 15+ |
| 移动浏览器 | iOS Safari 15+, Android Chrome 90+, 微信内置浏览器 |

### 3.3 安全需求

| 安全领域 | 具体需求 |
|---------|---------|
| 传输安全 | 全站HTTPS，HSTS启用，TLS 1.2+ |
| 认证安全 | JWT签名使用RS256，accessToken 2小时过期，refreshToken 7天过期 + 旋转刷新；密码bcrypt加盐哈希（cost factor >= 12） |
| 输入安全 | 所有用户输入经过验证和清洗（XSS过滤、SQL注入防护由TypeORM参数化查询保证）；文件上传类型白名单 + 大小限制 |
| 接口安全 | API限流（普通用户60次/分钟/IP，AI对话10次/分钟/用户）；CORS白名单配置；CSRF Token保护 |
| 数据安全 | 敏感数据（密码哈希、手机号）加密存储；数据库访问IP白名单；定期数据备份（每日全量 + 每小时增量） |
| 隐私安全 | 用户心理健康数据标记为敏感个人信息，加密存储；用户数据删除为不可逆物理删除（或脱敏保留）；数据导出需二次身份验证 |
| 日志安全 | 安全事件日志记录（登录失败、权限变更、风险检测）；日志不记录明文密码和完整Token；日志保留180天 |
| 风控安全 | 35+高危词 + 15+中危词库实时扫描；AI输出内容二次过滤；风险升级追踪（低→中→高）；管理员人工复核机制 |

### 3.4 易用性需求

| 需求项 | 具体要求 |
|-------|---------|
| 首次使用引导 | 未登录用户在首页看到项目介绍和功能概览，引导注册或登录 |
| 导航清晰度 | 底部导航栏最多5个核心入口（首页/AI对话/日记/?/我的），所有功能可在3次点击内触达 |
| 操作反馈 | 所有用户操作在300ms内提供视觉反馈（按钮状态变化、加载中动画、成功/失败提示） |
| 错误处理 | 所有错误以用户友好的语言提示（非技术术语），并提供明确的下一步行动建议 |
| 表单设计 | 长表单分步骤展示；必填项明确标注；实时输入校验和提示；错误信息紧邻对应字段 |
| 加载状态 | 所有异步操作显示骨架屏或加载动画；长内容逐批加载（分页或无限滚动） |
| 响应式设计 | 优先移动端设计，所有页面在手机端布局完整可用 |
| 可访问性 | 关键交互元素支持键盘导航；图片提供alt文本；颜色对比度满足WCAG AA级标准 |
| 内容可读性 | 正文最小字号16px，行高1.6—1.8；段落间适当留白；避免大段密集文字 |
| 品牌一致性 | 全站统一使用治愈系设计语言（雾松青/柔烟蓝/奶杏白 三色体系 + 大圆角 + 温暖圆润字体） |

**接口需求**：

- 分页响应格式：`{ list: [], total: number, page: number, pageSize: number, totalPages: number }`
- 错误响应格式：`{ code: number (非200), message: string, error: string (可选), timestamp: string }`

## 二、已妥善处理

| 序号 | 安全项 | 分类 | 实现方式 | 状态 |
|------|--------|------|---------|------|
| 1 | 密码加密存储 | 认证安全 | `UserService.register()` / `changePassword()` / `resetPassword()` 中使用 `bcryptjs.hash(password, 10)` (10轮salt) 加密存储密码。`User.entity.ts` 中字段名为 `passwordHash` 而非 `password`。`getProfile()` 使用 `select` 排除 `passwordHash` 字段 | ✅ 安全 |
| 2 | JWT密钥环境变量管理 | 敏感信息保护 | JWT 密钥通过 `ConfigService.get('JWT_SECRET')` 从 `.env` 文件读取，不硬编码在代码中。`JwtAuthGuard` (`server/src/common/guards/jwt-auth.guard.ts`) 使用 `jwtService.verifyAsync()` 验证 token 签名 | ✅ 安全 |
| 3 | API Guard 保护 | 授权控制 | 全局 `JwtAuthGuard` 保护所有端点，仅通过 `@Public()` 装饰器标记的端点（如 login/register/nav）可匿名访问。`RolesGuard` (`server/src/common/guards/roles.guard.ts`) 支持角色级别权限控制。前端 `router.beforeEach` 基于 localStorage token 进行路由守卫 | ✅ 安全 |
| 4 | 高危词实时检测 | 内容安全 | `RiskControlService.analyzeRisk()` 在每条用户消息发送前执行：先检查高危正则模式(HIGH_RISK_PATTERNS 含 10 条模式匹配如 `/不想.*活了?/`), 再检查高危关键词(HIGH_RISK_WORDS 含 35+ 词如'自杀'/'自残'/'跳楼'), 然后检查中危词(26词)和中危正则(10条)。高危(RiskLevel=2)时自动返回危机干预消息（含心理援助热线 400-161-9995）并通知管理员 | ✅ 安全 |
| 5 | 密码重置验证 | 认证安全 | `forgotPassword()` 向已注册手机发送 6 位数字验证码，验证码存入 Redis 5 分钟有效期。`resetPassword()` 验证码比对后重置密码并删除 Redis key。验证失败抛出 `BadRequestException` | ✅ 安全 |
| 6 | XSS防护 | 输出编码 | **前端层面：** Vue 3 模板引擎默认对 `{{ }}` 插值表达式进行 HTML 转义。**HTTP 头层面：** NestJS 默认设置 `X-Content-Type-Options: nosniff`。**输入层面：** `ValidationPipe` 全局开启 `whitelist: true` + `forbidNonWhitelisted: true` | ✅ 安全 |
| 7 | CORS白名单 | API安全 | `main.ts` 中 `app.enableCors()` 通过环境变量 `ALLOWED_ORIGINS` 配置允许的来源域名（使用 `credentials: true` mode），不硬编码 `*` | ✅ 安全 |
| 8 | 请求限流 | API安全 | `ThrottlerModule` (`app.module.ts`) 全局配置每个 IP 每分钟最多 100 次请求 (`ttl: 60000, limit: 100`)。`ChatService` 额外通过 `CacheService.incrementWeeklyChatCount()` 实现每周对话次数统计（上限50次） | ✅ 安全 |

## 四、建议修复

### 4.1 API密钥安全加固（🟡 中优先级）

**问题描述：** `.env` 文件中的 DeepSeek API Key 以明文形式存储。虽然该文件已加入 `.gitignore` 避免泄露到代码仓库，但服务器本地存储仍存在泄露风险。

**影响范围：** `server/.env` → `server/src/shared/ai/ai.service.ts`

**修复方案：**
```typescript
// 建议在 AiService 构造函数中增加如下保护：
constructor(private configService: ConfigService) {
  this.apiKey = this.configService.get('AI_MODEL_API_KEY', '');
  // 日志中脱敏打印
  if (this.apiKey) {
    this.logger.log(`AI Model configured: ${this.modelName}, provider: ${this.provider}, key: ${this.apiKey.slice(0, 4)}****`);
  }
}
```
同时建议长期方案接入环境变量加密或密钥管理服务。

**预计工作量：** 2小时

### 4.2 文件上传安全加固（🟡 中优先级）

**问题描述：** `UploadModule` 已创建但缺少文件类型白名单、大小限制和安全文件名处理。

**影响范围：** `server/src/modules/upload/upload.service.ts`, `server/src/modules/upload/upload.controller.ts`

**修复方案：**
```typescript
// upload.service.ts
import { extname } from 'path';
import { v4 as uuid } from 'uuid';

private readonly ALLOWED_EXTENSIONS = ['.jpg','.jpeg','.png','.gif','.pdf','.docx','.xlsx','.mp3','.mp4'];
private readonly MAX_FILE_SIZE = 5 * 1024 * 1024; // 5MB

isFileAllowed(file: Express.Multer.File): boolean {
  const ext = extname(file.originalname).toLowerCase();
  return this.ALLOWED_EXTENSIONS.includes(ext);
}

sanitizeFilename(originalname: string): string {
  const ext = extname(originalname);
  return `${uuid()}${ext}`;
}
```

**预计工作量：** 3小时

### 5.2 综合评价

**安全状况：良好，综合风险等级：低。**

项目在认证安全（bcrypt加密 + JWT验证）、内容安全（6类词库+危机干预）、API防护（限流+CORS+ValidationPipe）方面做了扎实的工作。风控系统尤其值得肯定，3级风险分级+危机干预机制能够有效保护高危用户。

**现有安全优势：**
1. 密码使用 bcryptjs (saltRounds=10) 加密存储，符合 NIST SP 800-63B 标准
2. JWT 密钥从环境变量读取，未硬编码在代码中
3. 全局 Guard 保护所有 API，白名单模式更安全
4. 风控系统在每条用户消息发送前实时检测，高危内容自动触发危机干预
5. 密码重置流程包含手机验证码二次验证
6. ValidationPipe 启用 `whitelist` + `forbidNonWhitelisted` 防止参数污染

**需改进的3个关键点：**
1. API密钥存储加固（`.env` 文件权限控制 + 日志脱敏）
2. 文件上传增加类型/大小白名单校验
3. 验证码接口增加发送频率限制和错误次数锁定

**无高危漏洞发现，可以继续安全地推进 Phase 5-6 开发。**

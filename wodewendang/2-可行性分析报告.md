## 一、项目需求分析概况

### 1.1 核心需求概述

随着社会竞争加剧和生活节奏持续加快，国民心理健康问题日益突出，对专业、便捷、低成本心理健康服务的需求呈现爆发式增长。本项目——"心理健康AI助手"——旨在通过人工智能技术构建一个集AI对话陪伴、情绪追踪、日记反思、冥想放松、知识学习、心理测评、视频咨询、危机干预于一体的综合性数字心理健康平台，有效弥补传统心理服务供给不足、成本高昂、地域不均衡等痛点。

### 1.2 系统规模估算

- **目标用户规模**：首年5万注册用户，第三年达60万，服务覆盖全国
- **系统并发量**：设计峰值支持5,000用户同时在线，AI对话并发500路
- **数据规模**：年新增对话消息预估500万条，年新增情绪记录200万条，年新增日记50万篇
- **技术复杂度**：14个TypeORM实体，11个业务模块，3种数据库异构存储，约18—22万行源代码

### 1.3 可行性初步判断

经初步评估，从技术成熟度、团队能力、市场需求、政策环境、资金预算五个维度考量，本项目具备充分的可行性。

| 维度 | 可行性评估 | 置信度 |
|------|----------|--------|
| 技术可行性 | 所有核心技术（LLM API、前后端框架、数据库）均为成熟开源技术，无技术攻关风险 | 95% |
| 经济可行性 | 投资回收期18—24个月，第三年可实现全面盈利 | 85% |
| 操作可行性 | Web应用操作门槛低，用户端使用路径清晰，管理后台易于培训 | 90% |
| 法律可行性 | 严格遵守《个人信息保护法》《数据安全法》，无重大法律合规障碍 | 90% |
| 市场可行性 | 市场需求真实且持续增长，竞争格局存在差异化机会 | 88% |
| 进度可行性 | 12个月完整开发周期合理，各阶段任务明确可衡量 | 85% |

**综合判断**：项目可行，建议正式立项并启动第一阶段开发。

### 2.4 法律可行性

本项目在法律法规层面的可行性分析如下：

**（1）个人信息保护合规**：严格遵循《中华人民共和国个人信息保护法》（2021年11月1日施行）。心理健康数据属于敏感个人信息，平台将在获得用户"单独同意"后方可收集，并告知数据处理目的、方式和范围。用户拥有查阅、更正、删除、撤回同意、数据可携带等法定权利。

**（2）数据安全合规**：遵循《中华人民共和国数据安全法》（2021年9月1日施行）。建立数据分类分级管理制度，对敏感个人信息实施加密存储和访问权限控制；制定数据安全事件应急预案，在发生数据泄露时48小时内向主管部门报告。

**（3）医疗健康合规界限**：本项目定位为**心理健康陪伴和自助工具**，而非医疗诊断设备或在线诊疗服务。平台所有AI回复均注明"本内容由AI生成，仅供参考，不能替代专业心理诊断和治疗"。不涉及《互联网诊疗管理办法》规制的在线诊疗行为，不需要医疗机构执业许可证。

**（4）未成年人保护**：遵循《中华人民共和国未成年人保护法》和《儿童个人信息网络保护规定》。对于16岁以下用户的注册，需获得监护人同意；未成年人使用AI对话功能时启用额外的安全过滤策略；禁止收集与服务无关的未成年人个人信息。

**（5）知识产权合规**：项目使用的心理量表（PHQ-9、GAD-7等）均为国际公开发表的标准化工具，不涉及版权争议；知识库内容原创或经合法授权使用；AI生成的回复不主张版权，用户生成内容版权归用户所有。

**（6）行业标准与伦理**：参照《人工智能心理健康服务伦理指南》（中国心理学会）和WHO《数字心理健康干预指南》，建立AI伦理审查机制，确保AI对话不产生歧视性、操控性或有害内容。

### 3.3 核心模块设计

**模块一：用户认证与账户管理**（User Module）
- JWT双Token机制（accessToken 2小时 / refreshToken 7天），自动刷新
- 支持三种登录方式：密码登录 + 验证码登录 + 忘记密码重置
- 用户档案管理：昵称、头像、个性签名、隐私设置
- 账户安全：密码强度校验、异地登录提醒、账户注销与数据删除
- 用户数据导出：支持导出个人数据为JSON格式，满足数据可携带权要求

**模块二：AI对话陪伴**（Chat Module）
- 支持多会话管理，用户可创建/删除/切换对话会话
- SSE流式响应实现AI逐字输出体验，配合非流式回退确保稳定性
- System Prompt包含心理健康领域约束：共情但不取代专业治疗、不提供诊断、高危机情境自动转介
- 风控系统实时扫描对话内容，高危检测触发自动干预
- 对话历史持久化至MongoDB，支持全文搜索和导出

**模块三：情绪日记**（Diary Module）
- 富文本日记编辑器，支持文字、时间戳、心情标签
- AI自动分析日记情绪倾向，生成情绪摘要和积极建议
- 日记列表支持按时间、心情类型筛选和搜索
- 统计面板展示日记写作频率、情绪变化趋势
- 日记支持编辑、删除、公开/私密设置

**模块四：心情追踪**（Mood Module）
- 快速心情记录（1—10分量表 + 情绪标签选择）
- 心情记录可视化：日/周/月趋势图、情绪分布饼图
- 心情笔记：在记录心情时附加简短文字说明
- 心情统计：平均值、最高最低、波动趋势等统计分析

**模块五：冥想放松**（Meditation Module）
- 冥想引导音频/文本库，按主题分类（减压/睡眠/专注/正念/身体扫描）
- 冥想计时器，支持自定义时长
- 冥想历史记录：时长、频率、连续天数统计
- 冥想成就系统：鼓励用户建立冥想习惯

**模块六：心理知识库**（Knowledge Module）
- 按分类组织的心理健康知识文章库
- 全文搜索 + 分类浏览 + 热门推荐
- 文章互动：点赞、收藏、评论（支持嵌套回复）
- 评论系统集成风控检查，防止有害内容传播
- Markdown富文本文章渲染

**模块七：心理测评**（Questionnaire Module）
- 标准化心理量表（PHQ-9抑郁筛查、GAD-7焦虑筛查等）
- JSON题库架构，支持灵活扩展新量表
- 自动评分计算 + 结果分级 + AI生成个性化建议
- 测评历史记录和结果对比

**模块八：视频咨询**（Video Module）
- 房间创建/加入/管理
- 心理咨询场景的视频会话支持
- 为后续对接专业咨询师预留接口

**模块九：危机热线**（Hotline Module，内嵌于系统）
- 全国心理危机干预热线信息聚合
- 高危对话自动提供热线信息和一键拨打引导
- 紧急联系人设置功能（可选）

**模块十：用户中心**（User Settings Module）
- 个人资料管理、密码修改、隐私设置
- 通知管理：系统通知、互动通知、提醒通知
- 帮助中心、关于页面
- 数据导出和账户注销

**模块十一：管理后台**（Admin Module）
- 数据仪表盘：核心指标可视化
- 用户管理：列表、详情、状态管理（启用/禁用/删除）
- 文章管理：创建/编辑/发布/审核（Markdown编辑器）

### 6.2 建议

**核心建议**：

1. **立即启动，抢占窗口**：当前是数字心理健康市场的黄金发展期，AI能力成熟、政策环境友好、用户认知快速提升，建议尽快启动项目开发。

2. **分阶段投入，控制风险**：不建议一次性投入全部预算。Phase 1—2投入约50万元完成MVP，验证产品市场契合度后，再追加投入进入后续阶段。

3. **AI安全为第一优先级**：心理健康赛道的特殊性决定了AI安全性比功能丰富度更重要。建议在Phase 1就建立基础风控体系，并在每个Phase迭代强化。

4. **建立专业顾问团队**：邀请2—3名心理学领域专业人士（临床心理师、高校心理学教授、精神科医生）组成外部顾问团，为产品方向、内容质量、AI伦理提供专业指导。

5. **关注政策动态**：明确产品定位为"心理健康陪伴和自助工具"，避免越界进入"互联网诊疗"范畴。订阅卫健委、网信办等相关部门的政策更新，及时调整合规策略。

6. **开源策略**：将用户端部分UI组件和通用工具模块开源，吸引开发者社区关注和贡献，形成技术品牌影响力。

7. **用户隐私保护设计**：从产品设计阶段就贯彻Privacy by Design理念，将数据最小化收集、用户知情同意、数据可删除等原则融入产品DNA。

## 七、其他重要意见

### 7.1 技术选型补充建议

- **建议引入API网关**（如Kong或自研Nginx Lua脚本）统一管理API路由、限流、认证和日志，降低各模块的耦合度
- **建议引入消息队列**（如RabbitMQ或Bull基于Redis）处理异步任务，如：AI分析日记情绪（避免阻塞用户请求）、批量推送通知、定时报告生成
- **建议建立Feature Flag机制**，支持功能的灰度发布和A/B测试，降低上线风险
- **建议前端引入渐进式Web应用（PWA）**技术，支持离线使用和桌面快捷方式，提升移动端体验

### 7.2 心理伦理委员会

建议在项目启动时成立**AI心理健康伦理委员会**（3—5人），由心理学专家、AI技术专家、法学专家、用户代表组成，负责：
- 审核AI对话策略和System Prompt的安全性
- 评估新功能的心理伦理风险
- 处理用户投诉和伦理争议
- 定期发布透明度报告

### 7.3 数据开放与研究合作

项目积累的心理健康对话和行为数据（脱敏后）具有极高的科研价值。建议：
- 与高校心理学/计算机学科建立合作关系
- 建立数据脱敏和开放规则，在保护用户隐私的前提下支持学术研究
- 将研究发现反哺产品，形成"研究—改进—再研究"的良性循环

### 7.4 国际化和多语言支持

架构设计应预留国际化（i18n）能力。在中文市场站稳后，可考虑拓展东南亚、日韩等亚洲市场，这些地区在文化上与中文心理健康观念有一定的相通性，且数字心理健康市场同样快速增长。

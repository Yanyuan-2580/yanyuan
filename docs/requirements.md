# 需求分析报告

---

## 1. 业务需求

### 1.1 项目背景
本项目是一个 **AI 心理健康助手**，旨在为用户提供专业的心理健康服务，包括情绪疏导、心理测评、冥想疗愈等功能，帮助用户缓解压力、调节情绪、提升心理健康水平。

### 1.2 业务目标
- 提供便捷的在线心理健康服务
- 帮助用户进行情绪管理和压力缓解
- 建立用户心理健康档案，提供个性化服务
- 辅助专业心理咨询师进行服务

### 1.3 核心业务场景
| 场景 | 描述 | 优先级 |
|------|------|--------|
| 情绪自测 | 用户进行每日情绪记录和测评 | 高 |
| AI 倾诉 | 用户与 AI 进行心理倾诉和对话 | 高 |
| 冥想疗愈 | 用户进行冥想和放松训练 | 中 |
| 知识阅读 | 用户阅读心理健康相关文章 | 中 |
| 日记记录 | 用户记录心情日记 | 高 |
| 专业咨询 | 用户预约专业心理咨询师 | 低 |

### 1.4 业务流程

```
用户登录 → 情绪自测 → AI 对话疏导 → 冥想训练 → 知识学习 → 日记记录
```

## 2. 用户需求

### 2.1 用户角色定义

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| **普通用户** | 寻求心理健康帮助的个人 | 情绪疏导、压力缓解、心理测评 |
| **管理员** | 系统后台管理人员 | 用户管理、内容审核、数据统计 |
| **心理咨询师** | 专业心理咨询人员 | 预约管理、咨询记录、用户档案 |

### 2.2 用户画像

#### 普通用户画像
- **年龄**：18-45 岁
- **职业**：职场人士、学生、自由职业者
- **痛点**：工作压力大、焦虑、情绪波动、缺乏倾诉渠道
- **需求**：便捷的情绪调节工具、专业的心理指导、隐私安全保障

#### 管理员画像
- **年龄**：25-40 岁
- **职业**：系统管理员、运营人员
- **痛点**：用户管理复杂、内容审核繁琐、数据统计困难
- **需求**：高效的管理工具、清晰的数据报表、便捷的操作界面

### 2.3 用户需求优先级

| 优先级 | 用户需求 | 来源 |
|--------|----------|------|
| P0 | 安全登录和隐私保护 | 所有用户 |
| P0 | 情绪自测功能 | 普通用户 |
| P0 | AI 聊天疏导 | 普通用户 |
| P1 | 冥想疗愈功能 | 普通用户 |
| P1 | 心理健康知识阅读 | 普通用户 |
| P1 | 心情日记记录 | 普通用户 |
| P2 | 专业心理咨询预约 | 普通用户 |
| P0 | 用户管理 | 管理员 |
| P0 | 内容审核 | 管理员 |
| P1 | 数据统计分析 | 管理员 |

## 6. 接口需求（概要）

### 6.1 用户模块 API
| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/user/register` | POST | 用户注册 |
| `/api/user/login` | POST | 用户登录 |
| `/api/user/profile` | GET/PUT | 获取/更新用户信息 |
| `/api/user/password` | PUT | 修改密码 |

### 6.2 聊天模块 API
| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/chat/sessions` | GET | 获取会话列表 |
| `/api/chat/messages` | GET | 获取消息列表 |
| `/api/chat/send` | POST | 发送消息 |

### 6.3 日记模块 API
| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/diary` | GET/POST | 获取日记列表/创建日记 |
| `/api/diary/:id` | GET/PUT/DELETE | 获取/更新/删除日记 |

### 6.4 知识模块 API
| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/knowledge/articles` | GET | 获取文章列表 |
| `/api/knowledge/article/:id` | GET | 获取文章详情 |
| `/api/knowledge/like` | POST | 点赞文章 |
| `/api/knowledge/collect` | POST | 收藏文章 |

### 6.5 管理模块 API
| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/admin/login` | POST | 管理员登录 |
| `/api/admin/users` | GET | 获取用户列表 |
| `/api/admin/user/:id` | PUT/DELETE | 更新/删除用户 |
| `/api/admin/articles` | GET/POST | 获取/创建文章 |
| `/api/admin/article/:id` | PUT/DELETE | 更新/删除文章 |
